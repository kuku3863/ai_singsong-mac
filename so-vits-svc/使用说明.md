# AI Song Project Backup

## 项目说明
这是一个 `so-vits-svc` (4.1-Stable) 项目的备份。
用于歌声转换 (Singing Voice Conversion)。

## 备份内容
- **代码**: 所有的 Python 脚本 (`*.py`)。
- **配置**: `configs/` 目录下的配置文件。
- **预训练模型**: `pretrain/` 目录下的基础模型 (Hubert, RMVPE 等)。
- **依赖**: `requirements.txt`。

## 未备份内容 (Excluded)
根据您的要求，以下内容**未**包含在此备份中，以节省空间：
- `logs/`: 训练生成的模型权重 (Checkpoints)。
- `dataset/`: 预处理后的数据集。
- `dataset_raw/`: 原始数据集 (建议单独备份源文件)。
- `results/`: 推理生成的音频结果。

## 如何恢复/使用
1. **环境配置**:
   确保安装了 Python 3.8+。
   安装依赖:
   ```bash
   pip install -r requirements.txt
   ```
   *注意: 如果您使用的是 `venv`，请重新创建虚拟环境，因为复制的 `venv` 可能无法在不同路径下工作。*

2. **恢复数据**:
   如果您有之前的 `dataset_raw` 或 `logs`，请将其复制回项目根目录。

3. **运行 WebUI**:
   使用以下命令启动本地 WebUI:
   ```bash
   python webUI_local.py
   ```
   或者运行 `Start_WebUI_Desktop.bat` (如果适用)。

## 项目分析与建议
### 1. 代码结构
- 主要入口为 `webUI_local.py`，集成了音频切片、预处理、训练和推理功能。
- 配置文件位于 `configs/`，其中 `44k.json` 是主要的训练配置。

### 2. 改进建议
- **模块化**: `webUI_local.py` 文件较大，建议将来将其拆分为多个模块 (如 `ui_inference.py`, `ui_train.py`) 以便于维护。
- **错误处理**: 部分代码缺乏详细的错误日志，建议在 `logs/` 下增加运行日志文件。
- **硬编码路径**: 项目中存在一些硬编码路径 (如 `logs/44k`)，建议统一提取到配置文件中。

## 常用命令
- **训练**: `python train.py -c configs/44k.json -m 44k`
- **推理**: `python inference_main.py ...` (建议使用 WebUI)
